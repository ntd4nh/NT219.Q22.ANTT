# Contributing

## Workflow

1. Tạo branch mới từ `main`.
2. Commit nhỏ, rõ nghĩa.
3. Mở pull request và mô tả thay đổi + cách test.

## Branch naming

- `feat/<ten-tinh-nang>`
- `fix/<ten-loi>`
- `chore/<ten-cong-viec>`
- `docs/<ten-tai-lieu>`

## Commit naming (gợi ý)

- `feat: ...`
- `fix: ...`
- `docs: ...`
- `chore: ...`

## Pull request checklist

- [ ] Đã test local.
- [ ] Không commit secret (`.env`, key, token).
- [ ] Cập nhật tài liệu liên quan nếu có.
